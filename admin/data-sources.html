Public Data Sources

Where to get trail/route data and how to convert it to a GPX you can upload on the Import page. Tracks are never merged — split big datasets per trail before importing.

USFS Motor Vehicle Use Maps (MVUM)

National Forest motorized roads & trails. On the page look for “Motor Vehicle Use Map: Roads” and “Motor Vehicle Use Map: Trails.”

  • Source: data.fs.usda.gov/geodata/edw/datasets.php
  • Format: shapefile, one per National Forest
  • Convert:
ogr2ogr -f GPX output.gpx input.shp -nlt LINESTRING -dsco GPX_USE_EXTENSIONS=YES

BLM Recreation / Route Data

  • Source: gis.blm.gov/EGISDownload/LayerPackages/
  • Format: shapefile
  • Convert: same ogr2ogr command as above

Colorado Parks & Wildlife — OHV Trails

  • Source: cpw.state.co.us/learn/Pages/OHVTrails.aspx
  • Format: varies — sometimes GPX (upload directly), sometimes shapefile (convert with the ogr2ogr command above)

Utah State Parks — OHV

  • Source: stateparks.utah.gov/activities/ohv/
  • Format: varies — GPX directly, or convert a shapefile as above

OpenStreetMap (Overpass Turbo)

  • Source: overpass-turbo.eu
  • Format: run the query, then Export → GPX from the Overpass Turbo UI

Sample query — moto-legal trails inside the current bounding box:

[out:xml][timeout:25];
(
  way["highway"="track"]["motor_vehicle"!="no"]["motorcycle"!="no"]({{bbox}});
  way["highway"="path"]["motorcycle"="yes"]({{bbox}});
);
out body;
>;
out skel qt;

Attribution required. OSM-derived data must credit © OpenStreetMap contributors. When a trail comes from OSM, include that credit in the trail Description before publishing.

Installing ogr2ogr (GDAL)

  • macOS: brew install gdal
  • Windows: gdal.org/download.html

After converting, upload the resulting .gpx on the Import page, pick the track, fill in name/region/difficulty, and publish.
